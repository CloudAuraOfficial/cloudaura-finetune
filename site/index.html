Fine-Tuning with LoRA & DPO

A QLoRA Pipeline for Structured JSON Extraction from Qwen2.5-1.5B-Instruct

Abstract. We present a four-stage fine-tuning pipeline combining quantized low-rank adaptation (QLoRA) with Direct Preference Optimization (DPO) for structured JSON extraction. Using 4-bit NF4 quantization and LoRA rank 16, we train only ~1.2% of model parameters on a single NVIDIA T4 GPU (Google Colab free tier). SFT on general chat data degraded JSON output capability, but DPO preference alignment recovered and improved upon the base model: +7.7% Key F1, 32% lower latency, and 87.5% valid JSON rate. This demonstrates DPO's power to rescue task-specific behaviour after domain-mismatched SFT.

Section 1

Training Pipeline

Four-stage pipeline from base model to preference-aligned fine-tune.

Stage I

Data Preparation

Load 500 samples from ultrachat_200k, format as JSON extraction instructions. Prepare 285 DPO preference pairs from curated chosen/rejected examples.

Stage II

SFT with QLoRA

4-bit quantized base model + LoRA adapters. 1 epoch on 500 samples. Paged AdamW 8-bit optimizer with cosine schedule.

Stage III

DPO Alignment

Merge SFT adapter, apply fresh LoRA. Train on 285 preference pairs with beta=0.1 sigmoid loss for preference alignment.

Stage IV

Evaluation

8 JSON extraction prompts. Measure valid JSON rate, key F1, value accuracy, and inference latency across all variants.

Section 4

Discussion: Why LoRA + DPO?

Design rationale and efficiency considerations for production-constrained fine-tuning.

4.1 QLoRA Efficiency

Full fine-tuning a 1.5B model requires ~12GB VRAM and updates all parameters. QLoRA quantizes to 4-bit and trains only ~0.6% of parameters via low-rank adapters. Result: fits on a free T4 GPU with minimal quality loss versus full fine-tuning.

4.2 SFT vs DPO

SFT teaches the model what to output. DPO teaches it which output is better. In our experiment, SFT on general chat data catastrophically degraded JSON output (0% valid JSON). DPO then recovered the model to 87.5% valid JSON and improved Key F1 over the base. This demonstrates DPO's power as a corrective alignment stage.

4.3 Rank Selection

LoRA rank (r) controls adapter capacity. r=4 is minimal, r=64 approaches full fine-tuning. We use r=16 as the sweet spot: enough capacity for task-specific adaptation without overfitting on 500 samples. Alpha=32 (2x rank) provides stable gradients.

4.4 Quantization Tradeoff

NF4 quantization reduces model size by ~4x. The double quantization flag further compresses quantization constants. Compute stays in bfloat16 for numerical stability. Measured quality loss vs full precision: <1% on most benchmarks.
